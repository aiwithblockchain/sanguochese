# 三国象棋开发计划

> 基于 `DESIGN.md` v0.2 与 `AI_TECH.md` v0.1，将开发任务划分为若干阶段。每个阶段标注当前状态为「未开发」。本计划只列大块任务，细节随实现调整。

---

## 阶段总览

| 阶段 | 名称 | 目标 | 状态 |
|---|---|---|---|
| P1 | 规则引擎 | 三方棋盘逻辑闭环，可验证走法正确性 | ✅ 完成 |
| P2 | 棋盘渲染 | SpriteKit 画出三瓣棋盘，可视化确认几何 | 未开发 |
| P3 | 人人对战 | 玩家可在 UI 上走子，完整一局对弈 | 未开发 |
| P4 | 灭国吞并 | 三方胜负状态机跑通 | 未开发 |
| P5 | 人机对战 | AI 引擎接入，可单人 vs AI | 未开发 |
| P6 | AI 解说 | DeepSeek 角色化解说层 | 未开发 |
| P7 | 打磨与优化 | 难度调节、性能、体验 | 未开发 |

---

## P1 · 规则引擎

**目标**：纯逻辑层，无 UI，可单元测试，验证三方走法与传统象棋一致。

| 任务 | 说明 | 状态 |
|---|---|---|
| P1-1 棋盘坐标系统 | 三方地盘坐标定义、九宫范围、国界位置 | ✅ |
| P1-2 接线表 σ | 翻转对接 i→(10−i)，六组映射实现 | ✅ |
| P1-3 国界分叉路由 | 过国界时二选一方向的路由逻辑 | ✅ |
| P1-4 棋子模型与初始布局 | 三方各 16 子，传统阵形摆位 | ✅ |
| P1-5 传统走法生成 | 车/马/炮/象/士/帅/兵的传统走法 | ✅ |
| P1-6 分叉走法生成 | 过国界棋子沿两方向各生成走法 | ✅ |
| P1-7 合法性校验 | 走子前校验，过滤非法走法 | ✅ |
| P1-8 三方主帅互照 | 同时检查两方照面，借刀杀人判定 | ✅ |
| P1-9 单元测试 | 两两对局与传统象棋对照验证 | ✅ 24/24 通过 |

## 阶段依赖关系

```
P1 规则引擎 ──► P2 棋盘渲染 ──► P3 人人对战 ──► P4 灭国吞并
                                                       │
                                                       ▼
                                                   P5 人机对战 ──► P6 AI 解说
                                                       │
                                                       ▼
                                                   P7 打磨优化
```

- P1 是一切基础，必须先完成。
- P2/P3 依赖 P1，可串行。
- P4 依赖 P3（需要能走子才能结算）。
- P5 依赖 P4（AI 需理解完整规则含吞并）。
- P6 依赖 P5（解说针对实际对局）。
- P7 贯穿后期，可与其他阶段并行迭代。
